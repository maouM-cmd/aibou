# プロンプト・エンジニアリング 2026

AIを極限まで使いこなすための、プロ仕様のプロンプトテクニック。

## 1. 構造化プロンプト
AIは構造化されたテキスト（Markdown等）を最も正確にパースできる。
*   **Bad**: 「Reactでログイン画面作って。バリデーションも入れて。あ、デザインはモダンな感じで」
*   **Good**:
    ```markdown
    # 役割
    あなたはシニアフロントエンドエンジニアです。
    
    # ゴール
    Reactを用いたログイン画面の作成
    
    # 要件
    - 入力: Email, Password
    - バリデーション: Zodを使用
    - スタイリング: Tailwind CSS (モダン、ダークモード対応)
    ```

## 2. 思考の連鎖 (Chain of Thought)
AIに複雑なタスクを頼むときは、「回答を出す前に、まず思考プロセスを出力せよ」と指示する。
*   **メンターからのアドバイス**: プロンプトの最後に `<thinking>タグの中で、実装方針と想定されるエラーを列挙してから、コードを書いてください` と加えるだけで、バグの発生率が劇的に下がる。
