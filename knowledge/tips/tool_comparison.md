# AI ツール使い分けパターン

## ツール横断の知見

### ツール別の得意分野
- **Cline**: ファイル操作が多い開発作業、プロジェクト全体を横断する変更
- **Cursor**: インライン編集、コード補完、速く書きたい時
- **Antigravity (Gemini)**: 調査・分析・ドキュメント生成、大きな設計判断
- **Ollama (ローカル)**: コスト気にせず試行錯誤、機密コード、簡単な修正
- 1つのツールに全部やらせない。得意分野で使い分けるのがプロ
- 記録日: 2026-07-08
- カテゴリ: 戦略

### Ollama × Cline のローカル活用
- 重要な設計・複雑なコード → Claude API（精度重視）
- 簡単な修正・リファクタ → Ollama ローカル（速い＆無料）
- 機密コードの作業 → Ollama ローカル（データが外に出ない）
- テスト生成 → Ollama ローカル（パターン的作業）
- タスクの種類でモデルを切り替えるとコスト激減
- 記録日: 2026-07-08
- カテゴリ: 戦略

### 「記憶」は自分で管理する
- AI は毎回記憶がリセットされる
- プロジェクトルートに memory.md を置いて、解決済みの問題・設計判断・注意事項を書く
- 毎回最初に読ませると、過去の失敗を繰り返さなくなる
- 記録日: 2026-07-08
- カテゴリ: ワークフロー
