# Cloud Code / Gemini CLI 使いこなしTips

## 基本設定

### GEMINI.md を活用する（最重要）
- プロジェクトルートに GEMINI.md を置くと、AIが毎回自動で読み込む
- Cline の .clinerules、Antigravity の AGENTS.md と同じ役割
- コーディング規約、アーキテクチャの制約、AIの振る舞い指示を書く
- 過剰な記述は出力品質を低下させるため、簡潔かつ明確に
- 記録日: 2026-07-08
- カテゴリ: 設定

### settings.json の階層を理解する
- グローバル設定: ~/.gemini/settings.json
- プロジェクト設定: <project>/.gemini/settings.json
- プロジェクト設定がグローバルを上書きする
- チームでルールを共有する時はプロジェクト設定を使う
- 記録日: 2026-07-08
- カテゴリ: 設定

## チェックポイント機能

### checkpointing を有効にする
- settings.json の general.checkpointing.enabled を true にする
- 複雑なリファクタリングや実験的な変更を安全に行える
- 内部的に Git を活用してプロジェクト状態を保持
- /restore [tool_call_id] で特定の状態まで巻き戻し可能
- Cline にはない強力な安全ネット機能
- 記録日: 2026-07-08
- カテゴリ: 機能

## サブエージェント

### 組み込みサブエージェントを活用する
- codebase_investigator: コードベース分析に特化
- cli_help: CLI操作のヘルプ
- メインエージェントがタスク内容に応じて自動的に適切なサブエージェントに委任
- 並列実行もサポートされている
- 記録日: 2026-07-08
- カテゴリ: 機能

### カスタムエージェントを作る
- Markdown形式でカスタムエージェントを作成可能
- 特定のワークフロー（ファクトチェック、ドキュメント生成など）を効率化
- .gemini/agents/ ディレクトリに配置
- 記録日: 2026-07-08
- カテゴリ: 上級

## MCP（Model Context Protocol）

### MCPサーバーで機能拡張
- settings.json の mcpServers セクションでサーバーを登録
- 外部API、データベース、Google Workspace等にアクセスできるようになる
- /mcp reload で設定の再読み込みが可能
- エージェントがターミナルを超えて外部サービスを直接操作できる
- Cline の MCP 対応と同じ仕組み（互換性あり）
- 記録日: 2026-07-08
- カテゴリ: 上級

## 効率化テクニック

### @ 参照でファイルを明示的に渡す
- @ファイル名 で特定のファイルをコンテキストに追加
- AIに「どのファイルが関係あるか推測させる」より精度が高い
- 大規模リポジトリでは特に効果的
- 画像ファイルも @ で渡せる
- 記録日: 2026-07-08
- カテゴリ: プロンプト

### カスタムスラッシュコマンドを定義する
- .gemini/commands/ に .toml ファイルを作成
- 独自の自動化コマンドを定義できる
- よく使うワークフローをコマンド化すると効率的
- 例: /test, /deploy, /review などを独自定義
- 記録日: 2026-07-08
- カテゴリ: 上級

### シェルモードを使う
- ! プレフィクスでシステムシェルにコマンドを直接送れる
- 例: !git status, !npm test
- AIの解釈を挟まずに直接実行したい時に便利
- 記録日: 2026-07-08
- カテゴリ: ワークフロー

### 非対話モードでCI/CDに組み込む
- --prompt オプションまたは標準入力（パイプ）でヘッドレス実行
- スクリプトやCI/CDパイプラインに組み込める
- 例: echo "テストを実行して結果を要約して" | gemini
- 記録日: 2026-07-08
- カテゴリ: 上級

### コンテキストの管理（重要）
- 会話が長くなると応答が遅く・精度が低下する
- 作業が一段落したら /clear でリセット
- 新しいタスクは新しいセッションで始める
- GEMINI.md があれば、リセットしてもプロジェクトの文脈は保持される
- 記録日: 2026-07-08
- カテゴリ: ワークフロー

## Cline との比較・使い分け

### Cloud Code が得意なこと
- Google Cloud のインフラ管理との連携
- 大きなコンテキストウィンドウを活かした分析
- チェックポイント機能による安全な変更
- MCP サーバーとの統合
- 記録日: 2026-07-08
- カテゴリ: 戦略

### Cline が得意なこと
- VS Code との深い統合（エディタ内で完結）
- Plan/Act モードの使い分け
- ファイル操作の細かい制御
- 記録日: 2026-07-08
- カテゴリ: 戦略

### ハイブリッド運用のコツ
- Cloud Code: リサーチ、大規模分析、インフラ操作
- Cline: ファイル編集中心の開発作業
- Ollama: コスト気にせず実験、機密コード
- 同じプロジェクトで複数ツールを使い分けるのが最強
- GEMINI.md と .clinerules に同じ内容を書いておくと一貫性が保てる
- 記録日: 2026-07-08
